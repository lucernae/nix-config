# nix-linux-builder

A remote aarch64 Linux builder (VPS/cloud) used as a Nix remote build machine
and binary cache server.

## Overview

- **Hostname**: `nix-linux-builder`
- **Architecture**: `aarch64-linux`
- **Emulates**: `x86_64-linux` via binfmt (for cross-compilation)
- **Services**: nix-serve (binary cache), nginx, tailscale, docker, SSH, fail2ban
- **Binary cache URL**: `https://nix-cache.maulana.id`

## Bootstrap: First-Time Remote Build & Activation

The machine is provisioned via [nixos-infect](https://github.com/elitak/nixos-infect)
or a fresh NixOS install. Once SSH access is available as `root`, use the steps
below from your local machine to deploy the NixOS configuration remotely.

### Prerequisites

1. SSH access to `root@nix-linux-builder` (or the machine's IP/hostname).
2. The remote host is reachable (directly or via Tailscale).
3. Local machine has Nix with flakes enabled.

### Step 1 — Copy SSH keys (if not already done)

```bash
ssh-copy-id root@nix-linux-builder
```

### Step 2 — Build and deploy from local machine

Use `nixos-rebuild` with `--target-host` to build locally and activate on the remote:

```bash
nixos-rebuild switch \
  --flake ~/.config/nix-config#nix-linux-builder \
  --target-host root@nix-linux-builder \
  --build-host localhost
```

This builds the derivation on the local machine, copies the closure to the remote
via SSH, and activates the new configuration there.

> **Tip**: If local machine is x86_64 and the target is aarch64, you need a
> cross-compilation setup or an aarch64 build machine. Options:
> - Use another aarch64 machine as `--build-host`
> - Enable aarch64 emulation locally: add `extra-platforms = aarch64-linux` and
>   `binfmt-misc` support to your local NixOS config

### Step 3 — Verify

```bash
ssh root@nix-linux-builder -- nixos-version
ssh root@nix-linux-builder -- systemctl status nix-serve nginx tailscale
```

## Subsequent Deployments

After bootstrapping, rebuild and redeploy any time the config changes:

```bash
nixos-rebuild switch \
  --flake ~/.config/nix-config#nix-linux-builder \
  --target-host root@nix-linux-builder
```

## Binary Cache Setup

The machine exposes a Nix binary cache at `https://nix-cache.maulana.id`.

To use it from other machines, add to your NixOS or nix.conf:

```nix
nix.settings = {
  substituters = [ "https://nix-cache.maulana.id" ];
  trusted-public-keys = [ "<paste public key here>" ];
};
```

To get the public key from the server:

```bash
ssh root@nix-linux-builder -- cat /root/nix-serve/cache-pub-key.pem
```

To push builds to the cache from a local machine:

```bash
nix copy --to ssh://root@nix-linux-builder <store-path>
```
